## การเริ่มต้นใช้งาน

### ฉันจะเข้าสู่แผงควบคุมผู้ดูแลระบบได้อย่างไร?

เข้าสู่ระบบด้วยข้อมูลประจำตัวผู้ดูแลระบบของคุณ และคุณจะเห็นเมนูนำทางผู้ดูแลระบบโดยอัตโนมัติ ซึ่งมีส่วนต่างๆ สำหรับแดชบอร์ด ยานพาหนะ ผู้ใช้ การจอง และการตั้งค่า ผู้ดูแลระบบมีสิทธิ์เข้าถึงคุณลักษณะทั้งหมดของระบบ รวมถึงการจัดการยานพาหนะ การบริหารผู้ใช้ และการกำหนดค่าระบบ

### ฉันจะสร้างผู้ดูแลระบบคนแรกได้อย่างไร?

คุณสามารถสร้างผู้ดูแลระบบคนแรกผ่าน Firebase Console ไปที่ Authentication → Users คลิก "Add User" จากนั้นไปที่ Firestore Database → users collection และสร้างเอกสารผู้ใช้ที่มีฟิลด์ role ตั้งเป็น "admin" ตรวจสอบให้แน่ใจว่าได้รวมฟิลด์ที่จำเป็นทั้งหมด: name, email, phone, role และ suspended (ตั้งเป็น false)

### ระบบมีบทบาทผู้ใช้แบบไหนบ้าง?

ระบบรองรับบทบาทหลักสองแบบ:

- **ผู้ดูแลระบบ (Admin)**: เข้าถึงได้เต็มรูปแบบในการจัดการยานพาหนะ การจัดการผู้ใช้ การจอง และการตั้งค่า สามารถดูข้อมูลทั้งหมดและดำเนินการด้านการบริหารทั้งหมด
- **ครู (Teacher)**: เข้าถึงจำกัดสำหรับดูรถที่มีอยู่และทำการจอง ไม่สามารถเข้าถึงฟังก์ชันด้านการบริหารหรือดูข้อมูลของผู้ใช้อื่น

### ฉันควรรู้อะไรเกี่ยวกับสถาปัตยกรรมของระบบ?

ระบบจองรถนี้สร้างด้วย Firebase (ฐานข้อมูล Firestore, Authentication และ Hosting) โดยใช้ React กับ TypeScript ใช้แผน Spark ฟรีของ Firebase ซึ่งมีข้อจำกัดเฉพาะเจาะจงในการอ่าน/เขียนข้อมูลรายวัน พื้นที่เก็บข้อมูล และแบนด์วิดธ์ ระบบได้รับการออกแบบสำหรับองค์กรขนาดเล็กถึงกลางที่มีรูปแบบการใช้งานปานกลาง

### ข้อจำกัดหลักของระบบคืออะไร?

ระบบทำงานบนแผน Spark ฟรีของ Firebase ซึ่งมีข้อจำกัดรายวันเหล่านี้:

- **การอ่านฐานข้อมูล**: 50,000 ครั้งต่อวัน
- **การเขียนฐานข้อมูล**: 20,000 ครั้งต่อวัน
- **พื้นที่เก็บข้อมูล**: 1 GB ทั้งหมด
- **แบนด์วิดธ์**: 10 GB ต่อเดือนสำหรับ hosting

ระบบไม่รองรับการแจ้งเตือนทางอีเมล การแก้ไขข้อขัดแย้งแบบเรียลไทม์ การวิเคราะห์ขั้นสูง การอัปโหลดรูปภาพ หรือรายงานที่ซับซ้อน ได้รับการออกแบบสำหรับฟังก์ชันการจองรถหลักพร้อมคุณลักษณะการจัดการพื้นฐาน

## การจัดการผู้ใช้

### ฉันจะเพิ่มผู้ใช้ใหม่ในระบบได้อย่างไร?

เนื่องจากข้อจำกัดของ Firebase free tier คุณไม่สามารถเพิ่มผู้ใช้โดยตรงจากแผงผู้ดูแลระบบได้ แต่คุณจัดการการลงทะเบียนผู้ใช้ผ่านระบบอีเมลที่อนุญาต:

1. ไปที่ **ผู้ดูแลระบบ → ผู้ใช้ → อีเมลที่อนุญาต**
2. เพิ่มที่อยู่อีเมล (ทีละหนึ่งบรรทัด) ในพื้นที่ข้อความ
3. คลิก **"เพิ่มอีเมลแบบกลุ่ม"** เพื่อเพิ่มไปยังรายการอนุญาต
4. แจ้งให้ผู้ใช้ลงทะเบียนด้วยที่อยู่อีเมลเหล่านั้น
5. ผู้ใช้ใหม่จะมีบทบาท "teacher" เป็นค่าเริ่มต้น

### ฉันจะเลื่อนตำแหน่งผู้ใช้เป็น admin ได้อย่างไร?

เพื่อเปลี่ยนบทบาทของผู้ใช้จาก teacher เป็น admin:

1. ไปที่ **ผู้ดูแลระบบ → ผู้ใช้**
2. ค้นหาผู้ใช้และคลิก **"แก้ไข"**
3. เปลี่ยนบทบาทจาก "teacher" เป็น "admin"
4. บันทึกการเปลี่ยนแปลง

ผู้ใช้จะต้องออกจากระบบและเข้าสู่ระบบใหม่เพื่อดูคุณลักษณะของ admin

### ฉันจะระงับบัญชีผู้ใช้ได้อย่างไร?

เพื่อระงับบัญชีผู้ใช้:

1. ไปที่ **ผู้ดูแลระบบ → ผู้ใช้**
2. ค้นหาผู้ใช้ที่คุณต้องการระงับ
3. คลิก **"ระงับ"** หรือคลิก **"แก้ไข"** และเปลี่ยนสถานะการระงับ
4. ยืนยันการดำเนินการ

ผู้ใช้ที่ถูกระงับไม่สามารถเข้าสู่ระบบหรือเข้าถึงระบบได้ พวกเขาจะเห็นข้อความ "บัญชีถูกระงับ" เมื่อพยายามเข้าสู่ระบบ คุณไม่สามารถระงับบัญชี admin ของตัวเองได้

### ฉันจะจัดการรายการอีเมลที่อนุญาตให้ลงทะเบียนได้อย่างไร?

ระบบอีเมลที่อนุญาตควบคุมว่าใครสามารถลงทะเบียนได้:

1. ไปที่ **ผู้ดูแลระบบ → ผู้ใช้ → อีเมลที่อนุญาต**
2. ดูอีเมลที่อนุญาตปัจจุบันพร้อมสถานะ (รอดำเนินการ/ลงทะเบียนแล้ว)
3. เพิ่มอีเมลใหม่โดยป้อนในพื้นที่ข้อความ (ทีละหนึ่งบรรทัด)
4. คลิก **"เพิ่มอีเมลแบบกลุ่ม"** เพื่อเพิ่มอีเมลหลายรายการพร้อมกัน
5. ลบอีเมลโดยเลือกและใช้ตัวเลือกลบ

มีเพียงอีเมลในรายการนี้เท่านั้นที่สามารถสร้างบัญชีใหม่ในระบบได้

### ฉันสามารถลบบัญชีผู้ใช้ได้หรือไม่?

บัญชีผู้ใช้ไม่สามารถลบอย่างถาวรได้เพื่อรักษาความสมบูรณ์ของข้อมูลและประวัติการจอง ระบบได้รับการออกแบบเพื่อรักษาข้อมูลประวัติศาสตร์ไว้เพื่อการตรวจสอบ แทนที่จะ คุณสามารถ:

- ระงับบัญชีเพื่อป้องกันการเข้าถึง
- เก็บข้อมูลประวัติการจองไว้ให้สมบูรณ์
- รักษาเส้นทางการตรวจสอบที่เหมาะสม

### ฉันจะดูประวัติการจองของผู้ใช้ได้อย่างไร?

เพื่อดูประวัติการจองที่สมบูรณ์ของผู้ใช้รายใด:

1. ไปที่ **ผู้ดูแลระบบ → ผู้ใช้**
2. คลิกที่ผู้ใช้เพื่อดูหน้ารายละเอียดของพวกเขา
3. หน้ารายละเอียดผู้ใช้แสดงการจองทั้งหมดพร้อมตัวเลือกการกรอง
4. คุณสามารถจัดการการจองของพวกเขาได้โดยตรงจากหน้านี้

### จะเกิดอะไรขึ้นเมื่อฉันแก้ไขโปรไฟล์ของผู้ใช้?

เมื่อคุณแก้ไขโปรไฟล์ของผู้ใช้:

- การเปลี่ยนแปลงชื่อและหมายเลขโทรศัพท์จะมีผลทันที
- ที่อยู่อีเมลไม่สามารถเปลี่ยนแปลงได้จากแผงผู้ดูแลระบบ (ผู้ใช้ต้องเปลี่ยนเอง)
- การเปลี่ยนแปลงบทบาทต้องให้ผู้ใช้ออกจากระบบและเข้าสู่ระบบใหม่เพื่อให้มีผล
- คำค้นหาจะได้รับการอัปเดตโดยอัตโนมัติเพื่อฟังก์ชันการค้นหาที่ดีกว่า

## การจัดการยานพาหนะ

### ฉันจะเพิ่มรถใหม่ในยานพาหนะได้อย่างไร?

เพื่อเพิ่มรถใหม่:

1. ไปที่ **ผู้ดูแลระบบ → ยานพาหนะ**
2. คลิกปุ่ม **"เพิ่มรถ"**
3. กรอกข้อมูลที่จำเป็น:
   - **ป้ายทะเบียน**: ต้องไม่ซ้ำกัน
   - **รุ่น**: รุ่น/ยี่ห้อของรถ
   - **ที่นั่ง**: จำนวนผู้โดยสาร
   - **สี**: สีของรถ
   - **ปี**: (ทางเลือก) ปีที่ผลิต
   - **คำอธิบาย**: (ทางเลือก) รายละเอียดเพิ่มเติม
4. ตั้งสถานะเริ่มต้น (พร้อมใช้งาน, ซ่อมบำรุง, หรือไม่พร้อมใช้งาน)
5. คลิก **"บันทึก"** เพื่อเพิ่มรถ

รถจะพร้อมใช้งานทันทีสำหรับการจองหากสถานะตั้งเป็น "พร้อมใช้งาน"

### ฉันจะเปลี่ยนสถานะของรถได้อย่างไร?

เพื่ออัปเดตสถานะความพร้อมใช้งานของรถ:

1. ไปที่ **ผู้ดูแลระบบ → ยานพาหนะ**
2. ค้นหารถในรายการ
3. คลิก badge สถานะเพื่อเปลี่ยนแปลงอย่างรวดเร็ว หรือคลิก **"แก้ไข"** เพื่อการเปลี่ยนแปลงโดยละเอียด
4. เลือกจาก:
   - **พร้อมใช้งาน**: สามารถจองได้โดยผู้ใช้
   - **ซ่อมบำรุง**: ไม่พร้อมใช้งานสำหรับการจอง
   - **ไม่พร้อมใช้งาน**: ไม่พร้อมใช้งานอย่างถาวร

มีเพียงรถที่มีสถานะ "พร้อมใช้งาน" เท่านั้นที่จะปรากฏในตัวเลือกการจองของผู้ใช้

### ฉันจะดูการวิเคราะห์ยานพาหนะได้อย่างไร?

แดชบอร์ดผู้ดูแลระบบให้การวิเคราะห์ยานพาหนะ:

1. ไปที่ **แดชบอร์ดผู้ดูแลระบบ**
2. ดู **แผนภูมิสถานะยานพาหนะ** ที่แสดงการกระจายของรถตามสถานะ
3. ดูจำนวนรถที่พร้อมใช้งาน อยู่ในการซ่อมบำรุง และไม่พร้อมใช้งานแบบเรียลไทม์
4. ติดตามรูปแบบการใช้งานยานพาหนะตลอดเวลา

คุณยังสามารถดูประวัติการจองของรถแต่ละคันได้โดยไปที่ **Admin → Fleet → [รายละเอียดรถ]**

### ฉันสามารถแก้ไขรายละเอียดรถหลังจากเพิ่มแล้วได้หรือไม่?

ได้ คุณสามารถแก้ไขรายละเอียดรถทั้งหมดได้ตลอดเวลา:

1. ไปที่ **ผู้ดูแลระบบ → ยานพาหนะ**
2. ค้นหารถและคลิก **"แก้ไข"**
3. แก้ไขฟิลด์ใดก็ได้ (ป้ายทะเบียน, รุ่น, ที่นั่ง, สี, ปี, คำอธิบาย)
4. อัปเดตสถานะหากจำเป็น
5. บันทึกการเปลี่ยนแปลง

**หมายเหตุ**: การเปลี่ยนป้ายทะเบียนจะส่งผลต่อการจองในอดีตทั้งหมดที่อ้างอิงถึงรถคันนั้น

### ฉันจะลบรถออกจากยานพาหนะได้อย่างไร?

เพื่อลบรถ:

1. ไปที่ **ผู้ดูแลระบบ → ยานพาหนะ**
2. ค้นหารถและคลิก **"แก้ไข"**
3. มองหาตัวเลือก **"ลบรถ"**
4. ยืนยันการลบ

**สำคัญ**: การลบรถจะส่งผลต่อข้อมูลการจองในอดีต พิจารณาตั้งรถเป็น "ไม่พร้อมใช้งาน" แทนหากคุณต้องการรักษาประวัติไว้

### ข้อมูลอะไรบ้างที่ถูกติดตามสำหรับรถแต่ละคัน?

ระบบติดตาม:

- **ข้อมูลพื้นฐาน**: ป้ายทะเบียน, รุ่น, ปี, สี, ที่นั่ง, คำอธิบาย
- **สถานะ**: สถานะความพร้อมใช้งานปัจจุบัน
- **ประวัติการจอง**: การจองทั้งหมดในอดีตและอนาคต
- **ข้อมูลการใช้งาน**: รูปแบบการใช้งานและความถี่
- **คำค้นหา**: สร้างโดยอัตโนมัติเพื่อการค้นหาที่มีประสิทธิภาพ
- **ประทับเวลา**: วันที่สร้างและแก้ไขล่าสุด

## การจัดการการจอง

### ฉันจะดูการจองทั้งหมดได้อย่างไร?

เพื่อดูการจองทั้งหมดในระบบ:

1. ไปที่ **ผู้ดูแลระบบ → การจอง**
2. ดูตารางครอบคลุมพร้อมการจองทั้งหมด
3. ใช้ตัวกรองเพื่อจำกัดผลลัพธ์:
   - **สถานะ**: กรองตามรอดำเนินการ, ยืนยันแล้ว, ยกเลิกแล้ว ฯลฯ
   - **ช่วงวันที่**: กรองตามวันที่เริ่มต้น/สิ้นสุด
   - **ผู้ใช้**: ดูการจองสำหรับผู้ใช้เฉพาะ
   - **รถ**: ดูการจองสำหรับยานพาหนะเฉพาะ
4. เรียงลำดับตามคอลัมน์ใด ๆ (ผู้ใช้, รถ, วันที่, สถานะ ฯลฯ)

### ฉันจะอนุมัติหรือปฏิเสธการจองได้อย่างไร?

เพื่อประมวลผลคำขอการจอง:

1. ไปที่ **ผู้ดูแลระบบ → การจอง**
2. ค้นหาการจองที่มีสถานะ "รอดำเนินการ"
3. คลิกที่การจองเพื่อดูรายละเอียด
4. เลือกที่จะ:
   - **ยืนยัน**: อนุมัติการจอง
   - **ปฏิเสธ**: ปฏิเสธการจอง
   - **ยกเลิก**: ยกเลิกหากยืนยันแล้ว

ระบบจะตรวจสอบความขัดแย้งโดยอัตโนมัติเมื่อยืนยันการจอง

### ฉันจะจัดการการจองที่ขัดแย้งกันได้อย่างไร?

เมื่อเวลาการจองซ้อนทับกัน:

1. ระบบป้องกันผู้ใช้จากการเลือกช่วงเวลาที่ถูกจองแล้ว
2. คุณสามารถตรวจสอบความขัดแย้งที่อาจเกิดขึ้นในรายการการจองได้ด้วยตนเอง
3. หากเกิดความขัดแย้ง คุณสามารถ:
   - มอบหมายรถอื่นให้กับผู้ใช้รายหนึ่ง
   - ติดต่อผู้ใช้เพื่อเจรจาเวลาที่แตกต่างกัน
   - ยกเลิกการจองหนึ่งรายการและแจ้งให้ผู้ใช้ทราบ
   - ปฏิเสธการจองใหม่ที่ขัดแย้งกับที่มีอยู่

### สถานะการจองที่แตกต่างกันหมายถึงอะไร?

สถานะการจองบ่งบอกถึงสถานะปัจจุบัน:

- **รอดำเนินการ**: รอการอนุมัติจาก admin
- **ยืนยันแล้ว**: อนุมัติและใช้งานได้
- **ยกเลิกแล้ว**: ยกเลิกโดยผู้ใช้หรือ admin
- **รอยกเลิก**: ผู้ใช้ขอยกเลิก รอการอนุมัติจาก admin
- **ปฏิเสธ**: ปฏิเสธโดย admin

### ฉันจะจัดการการยกเลิกการจองได้อย่างไร?

สำหรับคำขอยกเลิก:

1. ไปที่ **ผู้ดูแลระบบ → การจอง**
2. มองหาสถานะ "รอยกเลิก"
3. ตรวจสอบคำขอยกเลิก
4. อนุมัติหรือปฏิเสธการยกเลิก
5. ระบบจะอัปเดตสถานะตามนั้น

ผู้ใช้อาจสามารถยกเลิกโดยตรงได้ขึ้นอยู่กับการตั้งค่าระบบของคุณ

### ฉันสามารถแก้ไขรายละเอียดการจองได้หรือไม่?

ได้ คุณสามารถแก้ไขการจอง:

1. ค้นหาการจองใน **ผู้ดูแลระบบ → การจอง**
2. คลิก **"แก้ไข"** ในการจอง
3. แก้ไข:
   - วันที่และเวลาเริ่มต้น/สิ้นสุด
   - ชื่อผู้ขับขี่
   - ความคิดเห็น/หมายเหตุ
4. บันทึกการเปลี่ยนแปลง

**หมายเหตุ**: การเปลี่ยนแปลงวันที่จะเรียกใช้การตรวจสอบการซ้อนทับอัตโนมัติ

### ฉันจะดูการวิเคราะห์การจองได้อย่างไร?

แดชบอร์ดให้การวิเคราะห์การจอง:

1. ไปที่ **แดชบอร์ดผู้ดูแลระบบ**
2. ดู **แผนภูมิการจองรายวัน** ที่แสดง:
   - จำนวนการจองตามวัน
   - การแบ่งสถานะ (ยืนยันแล้ว, รอดำเนินการ, ยกเลิกแล้ว)
   - แนวโน้มรายเดือน
3. นำทางระหว่างเดือนเพื่อดูรูปแบบในอดีต
4. ติดตามอัตราการอนุมัติการจองและรูปแบบการยกเลิก

## การตั้งค่าระบบ

### ฉันจะกำหนดนโยบายการจองได้อย่างไร?

เพื่อตั้งค่ากฎการจอง:

1. ไปที่ **ผู้ดูแลระบบ → การตั้งค่า**
2. กำหนดค่าการตั้งค่าหลักเหล่านี้:
   - **วันจองล่วงหน้า**: ผู้ใช้สามารถจองได้ไกลแค่ไหน
   - **การอนุมัติอัตโนมัติ**: การจองได้รับการยืนยันโดยอัตโนมัติหรือไม่
   - **การยกเลิกอัตโนมัติ**: ผู้ใช้สามารถยกเลิกได้โดยไม่ต้องอนุมัติ
   - **ระยะเวลาการจองสูงสุด**: ความยาวการจองสูงสุด
   - **เวลาการยกเลิกล่วงหน้า**: การแจ้งให้ทราบล่วงหน้าขั้นต่ำที่จำเป็นสำหรับการยกเลิก
   - **การจองพร้อมกันสูงสุด**: การจองที่ใช้งานได้กี่รายการต่อผู้ใช้
   - **การจองวันหยุดสุดสัปดาห์**: อนุญาตหรือจำกัดการจองวันหยุดสุดสัปดาห์

**หมายเหตุ**: การตั้งค่า 0 (ศูนย์) สำหรับการตั้งค่าประเภทตัวเลขจะปิดใช้งานข้อจำกัดนั้น ๆ

### ฉันจะตั้งค่าชั่วโมงทำการได้อย่างไร?

เพื่อกำหนดชั่วโมงการทำงาน:

1. ไปที่ **ผู้ดูแลระบบ → การตั้งค่า**
2. ตั้งค่า **ชั่วโมงทำการ**:
   - **เวลาเริ่มต้น**: เมื่อไหร่ที่การจองสามารถเริ่มได้
   - **เวลาสิ้นสุด**: เมื่อไหร่ที่การจองต้องจบ
3. กำหนดค่า **การตั้งค่าวันหยุดสุดสัปดาห์**:
   - เปิดใช้งานหรือปิดใช้งานการจองวันหยุดสุดสัปดาห์
   - ตั้งเวลาที่แตกต่างกันสำหรับวันหยุดสุดสัปดาห์หากจำเป็น

**หมายเหตุ**: ชั่วโมงทำการใช้สำหรับเวลาที่ฝ่ายสนับสนุนพร้อมให้บริการเท่านั้น ไม่ใช่สำหรับข้อจำกัดการจอง

### ฉันจะจัดการข้อมูลการติดต่อฝ่ายสนับสนุนได้อย่างไร?

เพื่อตั้งค่าการติดต่อฝ่ายสนับสนุน:

1. ไปที่ **ผู้ดูแลระบบ → การตั้งค่า**
2. กำหนดค่า **อีเมลฝ่ายสนับสนุน**:
   - เพิ่มหลายที่อยู่อีเมลฝ่ายสนับสนุน
   - ผู้ใช้จะเห็นเหล่านี้ในหน้าช่วยเหลือของพวกเขา
   - การแจ้งเตือนทางอีเมลสามารถส่งไปยังที่อยู่เหล่านี้ได้
3. อัปเดตข้อมูลบริษัทที่แสดงใด ๆ

### ฉันจะล้างข้อมูลการจองเก่าได้อย่างไร?

เพื่อจัดการขนาดฐานข้อมูลและอยู่ภายในข้อจำกัดของ Firebase:

1. ไปที่ **ผู้ดูแลระบบ → การตั้งค่า**
2. ใช้เครื่องมือ **ล้างข้อมูล**:
   - เลือกปีที่จะลบการจองจาก
   - ดูตัวอย่างว่าจะลบการจองกี่รายการ
   - ยืนยันการลบ
3. แนะนำให้ล้างข้อมูลเป็นประจำทุกปี

**สำคัญ**: สิ่งนี้จะลบการจองเก่าอย่างถาวรและไม่สามารถยกเลิกได้

### ระบบมีการแจ้งเตือนทางอีเมลหรือไม่?

ระบบไม่มีการแจ้งเตือนทางอีเมลสำหรับการเปลี่ยนแปลงสถานะการจองเนื่องจากข้อจำกัดของ Firebase free tier และไม่มีการเข้าถึง Cloud Functions มีเพียงอีเมล Firebase Auth พื้นฐานเท่านั้นที่พร้อมใช้งาน:

- **การลงทะเบียนผู้ใช้**: การยืนยันอีเมลอัตโนมัติ
- **การรีเซ็ตรหัสผ่าน**: อัตโนมัติผ่าน Firebase Auth

ผู้ใช้ต้องตรวจสอบสถานะการจองของพวกเขาด้วยตนเองในระบบ นี่คือข้อจำกัดของแผนฟรี

### ฉันจะจัดการการตั้งค่าภาษาได้อย่างไร?

ระบบรองรับภาษาอังกฤษและไทย:

1. ผู้ใช้สามารถเปลี่ยนความต้องการภาษาจากส่วนหัวแอป
2. ความต้องการภาษาจัดเก็บในโปรไฟล์ผู้ใช้
3. อินเทอร์เฟซทั้งหมดจะอัปเดตเพื่อสะท้อนภาษาที่เลือก
4. การตั้งค่า admin ใช้กับผู้ใช้ทั้งหมด แต่ความต้องการแต่ละบุคคลจะแทนที่

### ฉันจะเปิดใช้งานหรือปิดใช้งานการอนุมัติการจองอัตโนมัติได้อย่างไร?

เพื่อกำหนดค่าเวิร์กโฟลว์การอนุมัติ:

1. ไปที่ **ผู้ดูแลระบบ → การตั้งค่า**
2. ค้นหาการตั้งค่า **การจองอัตโนมัติ**
3. เปิดใช้งานสำหรับการอนุมัติอัตโนมัติหรือปิดใช้งานสำหรับการอนุมัติด้วยตนเอง
4. เมื่อปิดใช้งาน การจองทั้งหมดจะมีสถานะ "รอดำเนินการ" จนกว่าคุณจะอนุมัติ

### การตั้งค่าการเพิ่มประสิทธิภาพคืออะไร?

เพื่อเพิ่มประสิทธิภาพระบบ:

1. ติดตามการใช้งาน Firebase ของคุณใน Firebase Console
2. ใช้เครื่องมือล้างข้อมูลเป็นประจำ
3. พิจารณาการตั้งค่าเหล่านี้:
   - ลดจำนวนการจองพร้อมกันสูงสุดต่อผู้ใช้
   - ระยะเวลาการจองล่วงหน้าที่สั้นกว่า
   - การล้างข้อมูลเก่าอัตโนมัติ

## Firebase และรายละเอียดทางเทคนิค

### ข้อจำกัดของแผน Firebase Spark คืออะไร?

แผน Spark ฟรีมีข้อจำกัดรายวันเหล่านี้:

- **การอ่าน Firestore**: 50,000 ครั้งต่อวัน
- **การเขียน Firestore**: 20,000 ครั้งต่อวัน
- **การลบ Firestore**: 20,000 ครั้งต่อวัน
- **พื้นที่เก็บข้อมูล**: 1 GB ทั้งหมด
- **แบนด์วิดธ์ Hosting**: 10 GB ต่อเดือน

การใช้งานหนักสามารถเกินข้อจำกัดเหล่านี้ ทำให้เกิดการหยุดชะงักของบริการชั่วคราว

### ฉันจะตรวจสอบการใช้งานระบบได้อย่างไร?

เพื่อติดตามการใช้งาน Firebase:

1. ไปที่ Firebase Console (สอบถามติดต่อทางเทคนิคของคุณสำหรับการเข้าถึง)
2. ตรวจสอบแท็บ **การใช้งาน** สำหรับเมตริกแบบเรียลไทม์
3. ติดตามการดำเนินการอ่าน/เขียนรายวัน
4. ตั้งค่าการแจ้งเตือนสำหรับการเข้าใกล้ขั้นต่ำ

ในแผงผู้ดูแลระบบ คุณสามารถดูการใช้งานพื้นฐานผ่านจำนวนการจองและผู้ใช้

### จะเกิดอะไรขึ้นหากเราเกินขีดจำกัดของ Firebase?

เมื่อเกินขีดจำกัด:

- **การอ่าน/เขียน**: บริการจะไม่พร้อมใช้งานชั่วคราว
- **พื้นที่เก็บข้อมูล**: ไม่สามารถเพิ่มข้อมูลใหม่ได้
- **แบนด์วิดธ์**: เว็บไซต์อาจไม่สามารถเข้าถึงได้

คุณจะต้องรอให้ขีดจำกัดรีเซ็ต (รายวัน) หรืออัปเกรดเป็นแผนที่ต้องจ่ายเงิน

### ฉันจะเพิ่มประสิทธิภาพของระบบได้อย่างไร?

เพื่อลดการใช้งาน Firebase:

1. **การล้างข้อมูลเป็นประจำ**: ลบการจองเก่าเป็นประจำทุกปี
2. **จำกัดผู้ใช้พร้อมกัน**: หลีกเลี่ยงช่วงเวลาการใช้งานสูงสุด
3. **การค้นหาที่มีประสิทธิภาพ**: ใช้คำค้นหาที่เฉพาะเจาะจง
4. **การดำเนินการแบบแบทช์**: ประมวลผลการจองหลายรายการพร้อมกัน
5. **ตรวจสอบแดชบอร์ด**: อย่ารีเฟรชโดยไม่จำเป็น

### ข้อมูลอะไรที่เก็บไว้ใน Firebase?

ระบบเก็บ:

- **ผู้ใช้**: โปรไฟล์, ข้อมูลการรับรองความถูกต้อง, ความต้องการ
- **รถ**: ข้อมูลยานพาหนะ, สถานะ, คำอธิบาย
- **การจอง**: รายละเอียดการจอง, สถานะ, ประวัติ
- **การตั้งค่า**: การกำหนดค่าระบบ, กฎธุรกิจ
- **อีเมลที่อนุญาต**: รายการอนุญาตการลงทะเบียน

ข้อมูลทั้งหมดได้รับการเข้ารหัสและปลอดภัยผ่านกฎความปลอดภัยของ Firebase

### ฉันจะสำรองข้อมูลระบบได้อย่างไร?

Firebase ให้การสำรองข้อมูลอัตโนมัติ แต่คุณยังสามารถ:

1. ส่งออกข้อมูลผ่าน Firebase Console
2. ใช้แผงผู้ดูแลระบบเพื่อดูและบันทึกข้อมูลสำคัญ
3. ส่งออกรายชื่อผู้ใช้และรถเป็นประจำ
4. บันทึกการตั้งค่าระบบและการกำหนดค่า

### จะเกิดอะไรขึ้นหากระบบล่ม?

หากระบบไม่พร้อมใช้งาน:

1. ตรวจสอบ Firebase Console สำหรับสถานะบริการ
2. ยืนยันการเชื่อมต่ออินเทอร์เน็ต
3. ตรวจสอบว่าได้เกินขีดจำกัดรายวันหรือไม่
4. ติดต่อฝ่ายสนับสนุนทางเทคนิคหากปัญหายังคงอยู่

การหยุดชะงักส่วนใหญ่เป็นชั่วคราวและจะแก้ไขภายใน 24 ชั่วโมงเมื่อขีดจำกัดรายวันรีเซ็ต

## การแก้ไขปัญหา

### ทำไมผู้ใช้ไม่สามารถเข้าสู่ระบบได้?

ตรวจสอบปัญหาทั่วไปเหล่านี้:

1. **สถานะบัญชี**: บัญชีถูกระงับหรือไม่?
2. **การยืนยันอีเมล**: ผู้ใช้ได้ยืนยันอีเมลของพวกเขาหรือไม่?
3. **ความสมบูรณ์ของโปรไฟล์**: ชื่อและหมายเลขโทรศัพท์ถูกกรอกหรือไม่?
4. **รหัสผ่าน**: ผู้ใช้ป้อนรหัสผ่านที่ถูกต้องหรือไม่?
5. **อีเมลที่อนุญาต**: อีเมลของพวกเขาอยู่ในรายการอนุญาตการลงทะเบียนหรือไม่?

### ทำไมรถไม่ปรากฏสำหรับการจอง?

ตรวจสอบการตั้งค่ารถเหล่านี้:

1. **สถานะ**: ต้องตั้งเป็น "พร้อมใช้งาน"
2. **ความขัดแย้ง**: ไม่มีการจองที่ซ้อนทับกันสำหรับเวลาที่ร้องขอ
3. **ความสมบูรณ์ของข้อมูล**: บันทึกรถไม่เสียหาย
4. **ปัญหาการค้นหา**: ลองรีเฟรชหน้าหรือล้างแคชเบราว์เซอร์

### ฉันจะจัดการปัญหาการยืนยันอีเมลได้อย่างไร?

สำหรับปัญหาการยืนยันอีเมล:

1. ตรวจสอบว่าที่อยู่อีเมลถูกต้องในโปรไฟล์ของผู้ใช้
2. ให้ผู้ใช้ตรวจสอบโฟลเดอร์สแปม/ขยะ
3. ให้ผู้ใช้ลองส่งอีเมลยืนยันอีกครั้งจากหน้าโปรไฟล์ของพวกเขา
4. ยืนยันว่าบริการอีเมล Firebase ทำงานอยู่
5. ตรวจสอบว่าโดเมนอีเมลกำลังบล็อกอีเมล Firebase หรือไม่

### ฉันควรทำอย่างไรหากระบบดูช้า?

ปัญหาประสิทธิภาพของระบบสามารถเกิดจาก:

1. **ขีดจำกัด Firebase**: ตรวจสอบว่าขีดจำกัดรายวันกำลังเข้าใกล้หรือไม่
2. **การเชื่อมต่ออินเทอร์เน็ต**: ยืนยันการเชื่อมต่อเครือข่าย
3. **ปัญหาเบราว์เซอร์**: ลองเบราว์เซอร์อื่นหรือล้างแคช
4. **การใช้งานสูง**: ผู้ใช้หลายคนเข้าถึงพร้อมกัน
5. **ชุดข้อมูลขนาดใหญ่**: การจองหรือผู้ใช้มากเกินไป

### ฉันจะแก้ไขข้อผิดพลาด "การอนุญาตถูกปฏิเสธ" ได้อย่างไร?

ข้อผิดพลาดการอนุญาตโดยทั่วไปบ่งชี้:

1. **บทบาทผู้ใช้**: ผู้ใช้ไม่มีสิทธิ์ admin
2. **เซสชันหมดอายุ**: ผู้ใช้ต้องออกจากระบบและเข้าสู่ระบบใหม่
3. **บัญชีถูกระงับ**: บัญชี admin ถูกระงับ
4. **กฎ Firebase**: กฎความปลอดภัยกำลังบล็อกการดำเนินการ

### ทำไมผู้ใช้ไม่ได้รับการแจ้งเตือนสถานะการจอง?

ระบบไม่ส่งการแจ้งเตือนทางอีเมลสำหรับการเปลี่ยนแปลงสถานะการจอง นี่คือข้อจำกัดของ Firebase free tier และขาดการเข้าถึง Cloud Functions ผู้ใช้ต้องตรวจสอบสถานะการจองของพวกเขาด้วยตนเองในระบบโดย:

1. เข้าสู่ระบบในบัญชีของพวกเขา
2. ดูประวัติการจองของพวกเขา
3. ตรวจสอบสถานะของการจองของพวกเขา

มีเพียงอีเมล Firebase Auth (การยืนยันและการรีเซ็ตรหัสผ่าน) เท่านั้นที่ส่งโดยอัตโนมัติ

### ฉันจะจัดการการเสียหายหรือสูญหายของข้อมูลได้อย่างไร?

สำหรับปัญหาเกี่ยวกับข้อมูล:

1. **การเปลี่ยนแปลงล่าสุด**: ตรวจสอบว่าการแก้ไขล่าสุดเป็นสาเหตุของปัญหา
2. **Firebase Console**: มองหาบันทึกข้อผิดพลาด
3. **รายงานผู้ใช้**: รวบรวมข้อมูลจากผู้ใช้ที่ได้รับผลกระทบ
4. **การกู้คืนข้อมูล**: ใช้เครื่องมือกู้คืนในตัวของ Firebase
5. **การคืนค่าระบบ**: อาจต้องการฝ่ายสนับสนุนทางเทคนิค

### ฉันควรทำอย่างไรหากการจองไม่บันทึก?

หากการจองไม่สามารถบันทึกได้:

1. **ขีดจำกัดฐานข้อมูล**: ตรวจสอบว่าขีดจำกัดการเขียนถูกเกินหรือไม่
2. **ข้อผิดพลาดการตรวจสอบ**: ตรวจสอบว่าฟิลด์ที่จำเป็นทั้งหมดถูกกรอก
3. **ความขัดแย้ง**: ยืนยันว่าไม่มีการจองที่ซ้อนทับกันอยู่
4. **ปัญหาเครือข่าย**: ตรวจสอบการเชื่อมต่ออินเทอร์เน็ต
5. **ปัญหาเบราว์เซอร์**: ลองเบราว์เซอร์อื่น

## ความปลอดภัยและการบำรุงรักษา

### ฉันจะจัดการสิทธิ์ผู้ใช้ได้อย่างไร?

ระบบใช้การควบคุมการเข้าถึงตามบทบาท:

1. **บทบาท Admin**: เข้าถึงเต็มรูปแบบไปยังคุณลักษณะและข้อมูลทั้งหมด
2. **บทบาท Teacher**: เข้าถึงจำกัดไปยังการจองและโปรไฟล์
3. **สถานะบัญชี**: บัญชีที่ถูกระงับไม่สามารถเข้าถึงอะไรได้
4. **การยืนยันอีเมล**: จำเป็นสำหรับบัญชีทั้งหมด
5. **ความสมบูรณ์ของโปรไฟล์**: จำเป็นสำหรับการเข้าถึงระบบเต็มรูปแบบ

### ฉันจะตรวจสอบการใช้งานระบบได้อย่างไร?

ติดตามกิจกรรมระบบผ่าน:

1. **แดชบอร์ด Admin**: สถิติการใช้งานพื้นฐาน
2. **กิจกรรมผู้ใช้**: รูปแบบการเข้าสู่ระบบและความถี่ของการจอง
3. **การใช้งานยานพาหนะ**: รูปแบบการใช้งานและความพร้อมใช้งานของรถ
4. **Firebase Console**: เมตริกทางเทคนิคโดยละเอียด
5. **การตรวจสอบข้อผิดพลาด**: ข้อผิดพลาดระบบและปัญหาประสิทธิภาพ

### คุณลักษณะด้านความปลอดภัยมีอะไรบ้าง?

ระบบรวมถึง:

1. **Firebase Authentication**: การรับรองความถูกต้องของผู้ใช้มาตรฐานอุตสาหกรรม
2. **การยืนยันอีเมล**: ป้องกันการสร้างบัญชีโดยไม่ได้รับอนุญาต
3. **การเข้าถึงตามบทบาท**: จำกัดการเข้าถึงตามบทบาทผู้ใช้
4. **กฎความปลอดภัย Firestore**: การควบคุมสิทธิ์ระดับฐานข้อมูล
5. **การเข้ารหัส HTTPS**: การส่งข้อมูลทั้งหมดได้รับการเข้ารหัส
6. **reCAPTCHA**: ป้องกันการใช้งานที่ผิดโดยอัตโนมัติ

### ฉันจะจัดการเหตุการณ์ด้านความปลอดภัยได้อย่างไร?

หากคุณสงสัยว่ามีปัญหาด้านความปลอดภัย:

1. **การดำเนินการทันที**: ระงับบัญชีที่ได้รับผลกระทบ
2. **การสืบสวน**: ตรวจสอบบันทึกกิจกรรมผู้ใช้
3. **เอกสาร**: บันทึกการค้นพบทั้งหมด
4. **การสื่อสาร**: แจ้งให้ผู้ใช้ที่ได้รับผลกระทบทราบหากจำเป็น
5. **การกู้คืน**: คืนค่าการดำเนินงานปกติและใช้การแก้ไข

### ฉันจะดำเนินการบำรุงรักษาเป็นประจำได้อย่างไร?

งานบำรุงรักษารายเดือน:

1. **การล้างข้อมูล**: ลบการจองเก่า
2. **การตรวจสอบผู้ใช้**: ตรวจสอบบัญชีที่ไม่ใช้งานหรือมีปัญหา
3. **การอัปเดตระบบ**: ตรวจสอบว่าส่วนประกอบทั้งหมดเป็นปัจจุบัน
4. **การตรวจสอบประสิทธิภาพ**: ติดตามรูปแบบการใช้งาน Firebase
5. **การยืนยันการสำรองข้อมูล**: ตรวจสอบว่าการสำรองข้อมูลทำงาน

### ฉันควรตรวจสอบอะไรเป็นประจำ?

ให้ความสนใจกับ:

1. **การใช้งาน Firebase**: การดำเนินการอ่าน/เขียนรายวัน
2. **กิจกรรมผู้ใช้**: รูปแบบการเข้าสู่ระบบและสถานะบัญชี
3. **รูปแบบการจอง**: แนวโน้มการจองและความขัดแย้ง
4. **ข้อผิดพลาดระบบ**: อัตราข้อผิดพลาดและข้อร้องเรียนของผู้ใช้
5. **ประสิทธิภาพ**: เวลาในการโหลดหน้าและการตอบสนอง

### ฉันจะวางแผนสำหรับการเติบโตของระบบได้อย่างไร?

เมื่อการใช้งานเพิ่มขึ้น:

1. **ตรวจสอบขีดจำกัด**: ติดตามการเข้าใกล้ขีดจำกัด Firebase
2. **เพิ่มประสิทธิภาพข้อมูล**: การล้างและเพิ่มประสิทธิภาพเป็นประจำ
3. **การจัดการผู้ใช้**: การเข้าร่วมผู้ใช้ที่มีประสิทธิภาพ
4. **การวางแผนกำลังการผลิต**: วางแผนสำหรับการอัปเกรดแผนที่ต้องจ่ายเงินที่อาจเกิดขึ้น
5. **คำขอคุณลักษณะ**: จัดลำดับความสำคัญของคุณลักษณะใหม่ตามการใช้งาน

## คุณลักษณะขั้นสูงและข้อจำกัด

### ทำไมคุณลักษณะเหล่านี้จึงไม่ได้ถูกพัฒนา?

แอพนี้ถูกพัฒนาในเวลาน้อยกว่า 2 สัปดาห์ บน Firebase free tier และด้วยทรัพยากรที่จำกัด ดังนั้น จึงไม่รวมคุณลักษณะขั้นสูงเช่น:

- การแจ้งเตือนทางอีเมลสำหรับการเปลี่ยนแปลงสถานะการจอง
- การแก้ไขข้อขัดแย้งแบบเรียลไทม์
- การวิเคราะห์และรายงานขั้นสูง
- เวิร์กโฟลว์การจัดการผู้ใช้ที่ซับซ้อน
- การอัปโหลดรูปภาพและสื่อที่หลากหลาย
- การรวมอัตโนมัติ
- เวอร์ชันแอปมือถือ
- การสนับสนุนแบบหลายผู้เช่า

การมุ่งเน้นอยู่ที่ฟังก์ชันหลักภายในขีดจำกัดของงบประมาณและเวลา

### มีวิธีแก้ไขข้อจำกัดที่ขาดหายไปหรือไม่?

สำหรับข้อจำกัดทั่วไป:

1. **ไม่มีการแจ้งเตือนทางอีเมล**: ผู้ใช้ต้องตรวจสอบสถานะการจองของพวกเขาด้วยตนเอง
2. **ไม่มีการอัปโหลดรูปภาพ**: ใช้คำอธิบายข้อความโดยละเอียดสำหรับรถ
3. **การวิเคราะห์จำกัด**: ส่งออกข้อมูลไปยังเครื่องมือภายนอกสำหรับการวิเคราะห์
4. **ไม่มีการอัปเดตแบบเรียลไทม์**: การรีเฟรชหน้าด้วยตนเองสำหรับข้อมูลล่าสุด
5. **การค้นหาพื้นฐาน**: ใช้หลายคำค้นหาสำหรับผลลัพธ์ที่ดีกว่า
6. **ไม่มีแอปมือถือ**: ใช้อินเทอร์เฟซเว็บที่ตอบสนองบนอุปกรณ์มือถือ

### ฉันจะขยายระบบได้อย่างไร?

การปรับปรุงที่อาจเกิดขึ้น:

1. **อัปเกรดเป็นแผน Firebase Blaze**: ลบข้อจำกัดการใช้งานและเปิดใช้งาน Cloud Functions
2. **เพิ่มการแจ้งเตือนทางอีเมล**: ใช้ Cloud Functions สำหรับการแจ้งเตือนอัตโนมัติ
3. **เพิ่มคุณลักษณะกำหนดเอง**: จ้างนักพัฒนาสำหรับฟังก์ชันเฉพาะ
4. **รวมเครื่องมือภายนอก**: เชื่อมต่อกับปฏิทินหรือระบบอีเมล
5. **การรายงานที่ปรับปรุง**: สร้างแดชบอร์ดแบบกำหนดเอง
6. **แอปมือถือ**: พัฒนาแอปพลิเคชันมือถือดั้งเดิม

### ขีดจำกัดความสามารถในการขยายของระบบคืออะไร?

ข้อจำกัดปัจจุบัน:

1. **ผู้ใช้**: ประมาณ 100-500 ผู้ใช้ที่ใช้งาน (ขึ้นอยู่กับการใช้งาน)
2. **รถ**: สูงสุด 50-100 คัน
3. **การจอง**: 200-1000 การจองต่อเดือน
4. **การเก็บข้อมูล**: 1 GB ทั้งหมด
5. **ผู้ใช้พร้อมกัน**: 10-20 ผู้ใช้พร้อมกัน

### ฉันจะขอคุณลักษณะใหม่ได้อย่างไร?

เพื่อแนะนำการปรับปรุง:

1. **เอกสารความต้องการ**: อธิบายฟังก์ชันที่ต้องการอย่างชัดเจน
2. **ประเมินผลกระทบ**: เข้าใจว่าจะส่งผลต่อผู้ใช้ปัจจุบันอย่างไร
3. **พิจารณาค่าใช้จ่าย**: ประเมินค่าใช้จ่ายการพัฒนาและการโฮสต์
4. **จัดลำดับความสำคัญของคุณลักษณะ**: มุ่งเน้นที่การเพิ่มที่สำคัญที่สุด
5. **การปรึกษาทางเทคนิค**: หารือกับนักพัฒนา

### พิจารณาระยะยาวคืออะไร?

สำหรับการใช้งานที่ยั่งยืน:

1. **อัปเกรดแผน**: พิจารณาแผน Firebase ที่ต้องจ่ายเงินเมื่อการใช้งานเพิ่มขึ้น
2. **การอัปเดตเป็นประจำ**: เก็บส่วนประกอบระบบให้เป็นปัจจุบัน
3. **การฝึกอบรมผู้ใช้**: ตรวจสอบว่าผู้ใช้เข้าใจข้อจำกัด
4. **ขั้นตอนการจัดการข้อมูล**: ใช้ขั้นตอนการบำรุงรักษาเป็นประจำ
5. **กลยุทธ์การสำรองข้อมูล**: รักษาการสำรองข้อมูลที่เชื่อถือได้

### ทำไมระบบถึงมีข้อจำกัดเหล่านี้?

ข้อจำกัดการออกแบบสะท้อนถึง:

1. **งบประมาณ**: พัฒนาด้วยต้นทุนต่ำ
2. **เวลา**: ไทม์ไลน์การพัฒนาที่รวดเร็ว
3. **ทรัพยากร**: ทีมพัฒนาจำกัด
4. **โครงสร้างพื้นฐาน**: ข้อจำกัดของ Firebase free tier
